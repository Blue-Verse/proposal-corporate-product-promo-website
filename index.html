핵심 요구사항 분석

  • 사용자 페이지 — 회사 소개, 제품 홍보(이미지 노출), 공지 게시판(텍스트 기반 정보 전달). 회원가입/로그인/회원관리 기능은 범위에서 제외.
  • 관리자 페이지 — 홈페이지 내 텍스트·이미지를 직접 수정하는 사이트 관리, 게시판 글 작성·수정·삭제 관리.
  • 인프라 신규 구축 — OS, 웹 서버, DB 등 신규 호스팅 환경을 처음부터 구성. 기존 솔루션 환경에서 독립된 신규 웹서버 환경으로 구축.
  • 비기능 요구사항 — PC·모바일 브라우저에 대응하는 반응형 웹, SSL 보안 인증서 적용.
  • 산출물 — 기획 문서, 디자인 원본 파일, 소스 코드 원본, 관리자 매뉴얼.

제안 포인트

본 프로젝트는 단순 소개용 사이트이지만, 핵심은 ① 비개발자도 직접 콘텐츠를 운영할 수 있는 관리자(CMS)와 ② 안정적인 신규 인프라(서버/DB/SSL) 구축에 있습니다. 이 두 축을 중심으로 제안합니다.

  • 운영 자율성 확보 — 사이트의 텍스트·이미지와 공지 게시판을 관리자 페이지에서 직접 수정·관리하도록 구성하여, 오픈 이후 별도 개발 의뢰 없이 콘텐츠를 갱신할 수 있습니다.
  • 제품 홍보에 최적화된 구조 — 제품 소개·이미지 노출을 카드/갤러리 형태로 구성하고, Next.js SSR/SSG 기반 SEO 최적화(메타태그·sitemap·OG 태그)로 검색 노출과 공유에 강한 사이트를 제공합니다.
  • 독립 신규 인프라 구축 — OS·웹서버(Nginx)·DB를 신규 호스팅에 처음부터 세팅하고, 도메인 연결 및 SSL(Let's Encrypt/ACM)까지 일괄 적용합니다.
  • 반응형 일관성 — PC·태블릿·모바일에서 동일한 브랜드 경험을 제공하는 반응형 디자인 시스템을 구축합니다.
  • 턴키 일괄 수행 — 기획·디자인·개발·인프라·배포를 단일 책임으로 진행하여 커뮤니케이션 비용을 줄이고 일정 리스크를 최소화합니다.

PRD (Product Requirements Document)

프로젝트 요구사항을 정의하고 기능 범위를 명확히 합니다.

1. 프로젝트 배경 및 목적

회사와 제품을 소개할 수 있는 신규 홈페이지를 서버 인프라부터 새롭게 구축합니다. 기존 솔루션 환경에서 독립된 신규 웹서버 환경으로 전환하며, 비개발자도 콘텐츠를 직접 운영할 수 있는 관리자 기능과 PC·모바일 반응형, SSL 보안을 갖춘 단순 소개용 사이트를 목표로 합니다.

2. 사용자 정의

사용자 유형설명주요 기능
방문자(일반 사용자)회사·제품 정보를 탐색하는 외부 방문자 (로그인 불필요)회사 소개 열람, 제품 홍보 열람, 공지 게시판 열람
운영 관리자사이트 콘텐츠와 게시판을 운영하는 내부 담당자사이트 텍스트·이미지 수정, 게시글 작성/수정/삭제

3. 기능 요구사항

3.1 사용자 페이지

  • [필수] 회사 소개 — 자사 안내용 페이지 구성
  • [필수] 제품 홍보 — 자사 취급 제품 소개 및 이미지 노출 (목록/상세, 갤러리)
  • [필수] 공지 게시판 — 공지사항 등 텍스트 기반 정보 전달 (목록/상세/검색/페이징)
  • [제외] 회원가입·로그인·회원 관리 기능 없음

3.2 관리자 페이지

  • [필수] 사이트 관리 — 홈페이지 내 텍스트 및 이미지 수정 기능
  • [필수] 게시판 관리 — 게시판 글 작성, 수정, 삭제 기능
  • [필수] 이미지/파일 업로드 및 관리

3.3 인프라 구축

  • [필수] 서버 및 DB 세팅 — OS, 웹 서버, DB 등 신규 호스팅 및 환경 구축
  • [필수] 기존 솔루션 환경에서 독립된 신규 웹서버 환경 구성
  • [필수] 도메인 연결 및 배포

4. 비기능 요구사항

항목요구사항
반응형PC 및 모바일 브라우저 환경에 대응하는 반응형 웹페이지
보안/인증SSL 보안 인증서 적용 (HTTPS 전 구간)
SEO검색엔진 노출 최적화 (메타태그, sitemap, OG 태그)
성능이미지 최적화 및 정적 페이지 캐싱으로 빠른 로딩
운영성관리자 페이지를 통한 비개발자 콘텐츠 자율 운영

5. 기술 제약사항

  • 기술 스택은 클라이언트 제안 요청 항목으로, 본 제안에서는 Next.js(FE) + NestJS(BE) + PostgreSQL + Nginx + AWS(EC2/RDS) 기반을 제안하며, 협의에 따라 조정 가능합니다.
  • 신규 인프라 구축이 과업에 포함되므로 호스팅 계정·도메인은 클라이언트 명의로 준비 협조가 필요합니다.

6. 범위 외 사항 (Out of Scope)

  • 회원가입·로그인·회원 관리 기능
  • 결제·전자상거래(장바구니/주문) 기능
  • 다국어(영문 등) 페이지 — 필요 시 별도 협의
  • 콘텐츠(원고·제품 사진) 제작 및 카피라이팅
  • 하자 보수 기간 이후의 운영 유지보수

EZ-Approve (기업용 전자결재 SaaS)

B2B SaaS · 기업용 소프트웨어 · 약 9주

프로젝트 개요

관리자 대시보드와 콘텐츠 편집 기능을 중심으로 한 기업용 포털형 SaaS. 사이트 콘텐츠·문서를 관리자가 직접 작성·수정하는 운영 구조를 구축.

핵심 기능

  • 관리자 대시보드 및 사이트 콘텐츠 관리(CMS)
  • 리치 텍스트 에디터 기반 문서 작성·수정·삭제
  • Next.js + NestJS 풀스택, Docker 기반 배포

본 프로젝트와의 유사점

본 과업의 핵심인 관리자 페이지(사이트 텍스트·이미지 수정 + 게시판 글 작성/수정/삭제)와 직접적으로 동일한 구조. 비개발자가 직접 콘텐츠를 운영하는 어드민 UX 설계 경험을 그대로 적용할 수 있습니다.

기술 스택

NestJS Next.js TypeScript MySQL Docker

Harmony Link (시니어 케어 관리 플랫폼)

B2B SaaS · 헬스케어 · 약 6개월

프로젝트 개요

웹·앱 기반 관리 플랫폼을 AWS 클라우드 인프라부터 신규로 구축. 멀티플랫폼 반응형 웹과 안정적 서버/DB 환경을 일괄 구성.

핵심 기능

  • Next.js/React 기반 반응형 웹 관리 화면
  • AWS CDK 기반 서버·DB 인프라 신규 구축 및 배포
  • 콘텐츠·로그 관리 시스템

본 프로젝트와의 유사점

본 과업의 서버/DB/인프라 신규 구축과 반응형 웹 요구를 모두 포함. 신규 호스팅 환경을 처음부터 세팅하고 배포까지 일괄 수행한 경험이 직접적으로 부합합니다.

기술 스택

Next.js NestJS React AWS CDK Docker
